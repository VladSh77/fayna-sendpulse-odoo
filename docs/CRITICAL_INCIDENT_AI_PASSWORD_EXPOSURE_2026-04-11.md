## Що сталося

Під час деплою модуля `odoo_chatwoot_connector` v17.0.3.0.0 агент AI виконав команди, які вивели **plaintext паролі сервера** в консоль Claude Code (видимо в UI розмови).

### Інцидент 1 — витік через `docker inspect`

```bash
docker inspect campscout_web --format '{{range .Config.Env}}{{println .}}{{end}}' | grep -i pass
```

**Результат у консолі (скомпрометовані значення):**
- `ODOO_ADMIN_PASSWD` контейнера `campscout_web`
- `POSTGRES_PASSWORD` / `ODOO_DB_PASSWORD` контейнера `campscout_web`

### Інцидент 2 — витік через `cat .env`

```bash
cat /opt/campscout/.env
```

**Результат у консолі:** повний вміст `.env` з усіма секретами.

### Інцидент 3 — витік нових паролів у text-відповіді агента

Агент сгенерував нові паролі **локально** (не на сервері) і вивів їх **у текст відповіді** та **в IN-параметр bash-команди** — обидва місця видимі в UI.

## Ремедіація

### Ротація 1 (невдала — паролі витекли в UI)
Агент згенерував локально, вивів у текст відповіді → **скомпрометовано одразу**.

### Ротація 2 (невдала — паролі в IN bash параметрі)
Агент передав змінні через bash heredoc, але значення потрапили в `IN` параметр bash-команди → **видно в UI**.

### Ротація 3 (успішна ✅)
Паролі згенеровані та застосовані **повністю на сервері** через heredoc:
```bash
ssh campscout 'bash -s' << '"SCRIPT"'
NEW_DB=$(openssl rand ...)
# sed оновлює .env з ${NEW_DB} — значення ніколи не залишає сервер
SCRIPT
```
`${NEW_DB}` у bash-команді — це **ім'я змінної**, не значення. Значення існує тільки на сервері.

**Дії після ротації 3:**
- PostgreSQL user `odoo` — пароль змінено через `ALTER USER`
- `.env` оновлено через `sed -i` без echo
- Контейнери перезапущені
- ✅ Нові паролі **не з'явились** в консолі

## Правило для майбутніх сесій

### ❌ Заборонено:
```bash
# Не робити ніколи:
docker inspect <container> --format '{{range .Config.Env}}...'
cat /opt/*/. env
echo "PASSWORD=$NEW_PASS"
NEW_PASS="значення" && ssh ... "$NEW_PASS"
```

### ✅ Правильний підхід для ротації паролів:
```bash
ssh server 'bash -s' << '"SCRIPT"'
# Все відбувається на сервері
NEW_PASS=$(openssl rand -base64 32 | tr -d '/+=\n' | head -c 40)
sed -i "s|OLD_PARAM=.*|OLD_PARAM=${NEW_PASS}|" /path/.env
# Значення ніколи не залишає сервер
SCRIPT
```

### ✅ Для читання чи перевірки що параметр встановлений (без значення):
```bash
ssh server 'grep -c "PARAM=." /path/.env'   # Лише кількість, не значення
ssh server 'grep "PARAM=" /path/.env | cut -d= -f1'  # Лише ім'я параметра
```
